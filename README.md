## 1. Описание проекта

**Doomlike** — 3D-шутер от первого лица в жанре arena FPS, вдохновлённый Doom Eternal, Quake и Serious Sam. Игрок сражается на замкнутой арене с волнами врагов, используя три вида оружия. Механика щитов у врагов создаёт ситуацию, в которой каждое оружие имеет свою незаменимую роль.

Цель проекта — реализовать полный цикл разработки игры: архитектуру кода, геймплейные системы, искусственный интеллект противников, VFX, UI и звук.

---

## 2. Геймплей

Игра разворачивается на замкнутой арене. Игрок начинает первую волну и должен уничтожить всех противников, после чего автоматически начинается следующая — всего три волны нарастающей сложности.

**Цель:** пережить все три волны, не потеряв все очки здоровья.

**Управление:**

| Действие | Клавиша |
|---|---|
| Движение | WASD |
| Прыжок | Space (одиночное нажатие) |
| Рывок | Space (удержание) |
| Стрельба | ЛКМ |
| Переключение оружия | 1 / 2 / 3 или колесо мыши |
| Лок-он | LeftShift (удержание) |

**Ресурсы:** здоровье и броня восстанавливаются пикапами, которые враги роняют при смерти с определённой вероятностью. Броня поглощает 50% входящего урона, пока не иссякнет.

**Победа / поражение:** после уничтожения всей третьей волны игра считается пройденной. Если здоровье игрока падает до нуля — поражение.

## 3. Ключевые механики

### 3.1 Три вида оружия

Каждое оружие имеет выраженную роль и незаменимо в конкретной ситуации:

| Оружие | Механика | Роль |
|---|---|---|
| **Rifle** | Hitscan, автоматический огонь, лёгкий разброс | Универсальное — эффективно против любых незащищённых врагов |
| **Shotgun** | Hitscan, 8 лучей по конусу, высокий урон на пеллет | Ближний бой — мгновенно уничтожает Melee-врагов вплотную |
| **PlasmaGun** | Снаряд со скоростью 40 м/с | Единственное оружие, способное перегреть и уничтожить щит |

Переключение — клавиши 1/2/3 или колесо мыши.

### 3.2 Рывок (Dash)

Нажатие Space при движении бросает игрока вперёд на 12 м/с в течение 0.18 с. На время рывка персонаж **неуязвим** — это позволяет уходить от атак в последний момент. Кулдаун — 1.2 с; HUD показывает радиальный индикатор заполнения.

### 3.3 Лок-он (Lock-On)

Удержание LeftShift при наличии врага в поле зрения «прилипляет» камеру к цели через Cinemachine. Прицел-ретикул рендерится поверх врага в UI-пространстве. При смерти цели лок-он автоматически снимается. Механика полезна для точного удержания прицела на быстро движущихся врагах.

### 3.4 Щиты и перегрузка

**ShieldedRangedEnemy** несёт энергетический щит:
- Пули (Rifle, Shotgun) поглощаются щитом полностью — урон равен нулю.
- Плазменные заряды накапливают `heat` в щите. При достижении максимального значения происходит **Overload**:
  - AoE-взрыв радиусом 5 м наносит урон всему в зоне поражения.
  - Визуальный бёрст: щит расширяется и угасает.
  - После перегрузки враг остаётся без защиты и уязвим для любого оружия.
- Цвет щита меняется от синего к красному по мере нагрева, сигнализируя об опасности взрыва.

## 5. Архитектура кода

Проект разбит на независимые модули (namespace-папки):

```
Assets/Scripts/
├── Core/       DamageInfo, IDamageable, Health, AudioOneShot, MusicPlayer
├── Player/     Controller, InputReader, Health, Dash, LockOn, визуальные компоненты
├── Weapons/    WeaponBase (abstract), Rifle, Shotgun, PlasmaGun, WeaponHolder
├── Enemies/    EnemyBase (abstract), MeleeEnemy, RangedEnemy, ShieldedRangedEnemy, EnemyShield
├── Pickups/    Pickup (abstract), HealthPickup, ArmorPickup, LootDropper
├── Spawning/   EnemySpawner, WaveDefinition (ScriptableObject)
└── UI/         HUDController, LockOnReticle, MainMenuController
```

**Ключевые архитектурные решения:**

- **`IDamageable` / `DamageInfo`** — единый контракт для нанесения урона. `DamageInfo` (struct) передаёт количество урона, тип (`Bullet / Plasma / Explosion`), источник, точку и нормаль попадания. Благодаря `in`-параметру нет выделения памяти.
- **`WeaponBase`** — абстрактный класс с общей логикой стрельбы (fire rate, боеприпасы, звук, VFX). Подклассы реализуют только `Fire(origin, direction)`.
- **`EnemyBase`** — абстрактный класс для NavMeshAgent-врагов с шаблоном состояний. Подклассы реализуют `UpdateBehavior()`.
- **`Pickup`** — абстрактная база с боббингом и вращением; подклассы реализуют `Apply(PlayerHealth)`.
- **`WaveDefinition`** — ScriptableObject с массивом {prefab, count}; волны редактируются без изменения кода.

## 6. Реализованные системы

### 6.1 Игрок

| Компонент | Функциональность |
|---|---|
| `PlayerController` | Движение (CharacterController), прыжок, mouse-look через новый Input System |
| `PlayerInputReader` | Фасад над `PlayerControls`; предоставляет события и текущие значения всем способностям |
| `PlayerHealth` | 100 HP + броня (поглощает 50% урона); события `Changed`, `Died`; звуковые подсказки при низком HP и смерти |
| `DashAbility` | Мгновенный рывок (12 м/с, 0.18 с); кулдаун 1.2 с; I-frames на время рывка |
| `LockOnSystem` | Поиск ближайшего врага в конусе перед камерой; переключение Cinemachine vcam; авто-отмена при смерти цели |
| `CameraBob` | Синусоидальное покачивание камеры при движении |
| `ViewModelSway` | Плавное смещение вьюмодели при движении мыши |

### 6.2 Враги

| Тип | Поведение | HP |
|---|---|---|
| **MeleeEnemy** | Преследует игрока, в 2 м — swipe-атака (кулдаун 1.2 с) | 30 |
| **RangedEnemy** | Держит дистанцию 8–14 м, стреляет медленным снарядом | 60 |
| **ShieldedRangedEnemy** | Как Ranged + `EnemyShield`; Bullet урон = 0 пока щит активен | 60 |

Навигация всех врагов — Unity NavMesh (запечено для арены).

### 6.3 Пикапы и лут

- **HealthPickup** — восстанавливает 25 HP.
- **ArmorPickup** — добавляет 25 брони.
- **LootDropper** — на событии `Killed` врага с настраиваемой вероятностью спавнит пикап.
- Все пикапы боббятся и вращаются; при подборе — SFX + VFX + уничтожение объекта.

### 6.4 Спавн волн

`EnemySpawner` слушает количество живых врагов. Когда волна полностью уничтожена — автоматически загружает следующую `WaveDefinition` и спавнит врагов на точках по периметру.

Три предустановленные волны:
1. Melee + Ranged
2. Melee + Ranged + ShieldedRanged
3. Melee + Ranged + ShieldedRanged (больше врагов)

### 6.5 UI / HUD

- Полоски HP и брони с плавным заполнением.
- Текущее оружие и количество патронов.
- Radial-индикатор кулдауна рывка.
- Crosshair и lock-on reticle (следует за целью в 2D).
- Счётчик волны и убийств.
- Красная вспышка экрана при получении урона.
- Главное меню с кнопками Play / Quit.

Адаптивность под разные разрешения — Canvas Scaler в режиме **Scale With Screen Size** (Reference 1920×1080).

### 6.6 VFX (VFX Graph)

| Эффект | Описание |
|---|---|
| `VFX_Rifle_Muzzle` | Короткая дульная вспышка для Rifle |
| `VFX_Plasma_Muzzle` | Заряженная вспышка для PlasmaGun |
| `VFX_BulletImpact` | Искры + дым при попадании пули |
| `VFX_PlasmaImpact` | Flash + shockwave ring + energy spread (3 суб-графа) |
| `VFX_ShieldExplosion` | Крупный AoE-взрыв при перегрузке щита |
| `VFX_ImpactSpark` | Общие искры при физическом контакте |

### 6.7 Звук

- **`AudioOneShot`** — статический helper: создаёт временный `AudioSource`, играет клип, уничтожает объект. Поддерживает pitch-variance и spatial blend (0 = 2D, 1 = 3D).
- **`MusicPlayer`** — фоновая музыка с кроссфейдом двух `AudioSource`; `DontDestroyOnLoad` между сценами; случайный или последовательный порядок треков.

### 6.8 Рендеринг и освещение

- URP **Volume Profile**: Bloom (threshold 1.1, intensity 0.6 — подчёркивает эмиссию плазмы и щитов), Vignette, Color Adjustments, Chromatic Aberration.
- Запечённое освещение (Mixed Directional Light + Light Probes для врагов).
- Reflection Probe в центре арены.

## 7. Итоги и выводы

За время проекта реализован полноценный arena FPS, включающий:

- **Три вида оружия** с разными механиками (hitscan vs. projectile).
- **Три типа врагов** с NavMesh-навигацией и простой машиной состояний.
- **Уникальную механику щитов**: принудительное переключение на плазмаган.
- **Способности игрока**: рывок с I-frames, лок-он через Cinemachine.
- **VFX Graph-эффекты** для всех ключевых игровых событий.
- **Волновой спавн** на основе ScriptableObject-данных.
- **Полноценный HUD** с адаптацией под разные разрешения.

Проект демонстрирует применение паттернов проектирования (интерфейсы, абстрактные классы, событийная модель), возможностей Unity 6 URP (VFX Graph, Cinemachine 3, AI Navigation 2) и принципов читаемого, расширяемого кода без избыточных абстракций.
